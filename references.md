## 2. 메인 페이지 구조 및 네비게이션

- 상단 네비게이션: 로고, 서비스 알아보기, 시작하기 버튼
- 주요 메시지: "듀티표의 마침표, 듀티메이트"
- 부메시지: "간호사 임무의 효율성과 공정성을 높이는 근무표 자동 생성 서비스"

**액션 버튼:**
- 서비스 알아보기 (Learn more)
- 시작하기 (Get started)
- 듀티메이트팀에게 커피 후원하기 (Support team with coffee)

### 2.2 Dutying (https://www.dutying.net)
**페이지 구성:**
- 상단 네비게이션 바: 로고 + 메뉴 항목 + 로그인 버튼
- 네비게이션 메뉴:
  - 웹 주요 기능 (Web Key Features)
  - 모바일 앱 주요 기능 (Mobile App Key Features)
  - 다운로드 (Download)
  - 문의하기 (Contact)
  - 로그인 (Login)

**메인 섹션:**
- 대제목: "근무표, 이제 더 간편하게!" (Now, duty schedules are easier!)
- 두 가지 플랫폼 설명:
  - **App**: 원청 관리자 모든 여정이 더 편리해지는 경험 제공 (General nurses)
  - **Web**: 근무표를 더 쉽고 빠르게 작성할 수 있도록 도움 (Head nurses/Managers)

**액션 버튼:**
- 근무표 작성 체험하기 (Try creating schedule)
- 근무표 만들기 (Create schedule)
- App 다운로드 (Google Play, App Store)

### 2.3 DutyPlanners (https://dutyplanners.com)
**페이지 구성:**
- 상단 배너: 신입순 20명 환영 + 파워드 참여 시 6개월 30% 할인 + 전담 매니저 지원
- 상단 네비게이션: 로고 + 가이드 + 블로그 + 무료로 시작하기 (오른쪽 정렬)

**메인 섹션:**
- 대제목: "공정하고 효율적인 간호사 스케줄 자동 생성 및 관리"
- 부제목: "복잡한 제약조건을 자동으로 처리해 3시간 작업을 1분으로 단축합니다. 간호사들의 만족도를 높이고, 관리자의 부담을 줄이는 듀티플래너입니다."
- 신뢰 요소: "상급종병원 특수훈련 5년차 간호사 & 상급종병원 30년차 병동 간호사와 함께 개발"
- 마케팅 메시지: "파워드 운영 중·추가 병원 모집 중"

**액션 버튼:**
- 무료로 시작하기 (Start for free)
- 방문 소개 신청하기 (Request hospital visit)

**스케줄 샘플 표시:**
- 실제 근무 스케줄 표 샘플 (D, E, N, O 등의 근무 유형 표시)

### 2.4 MakeDuty (https://makeduty.com)
**페이지 구성:**
- 헤더: 로고 (Make Duty) + 우측 상단 "듀티 작성하기" 버튼
- 페이지 제목: "간호사 근무표" (Nurse Duty Schedule)

**메인 섹션 - 근무 계획 기본 설정 (좌측 패널):**
- 총 간호사 수 입력
- 스케줄 작성 월 선택
- 근무 순서 설정 (토글 옵션들)

**메인 섹션 - 근무표 작성 규칙 설정 (우측 패널):**
- 최대 연속 근무일 수
- 최대 연속 야간근무(N) 수
- 일일 휴무일 수
- 평일별 각 근무 유형별 인원 수
- 주말별 각 근무 유형별 인원 수

## 3. 제공 서비스의 주요 기능 목록

### 3.1 DutyMate
**기본 기능:**
- ✅ 자동 근무표 생성 (Automatic duty schedule generation)
- ✅ 모바일 앱 (Mobile app for nurses)
- ✅ 웹 인터페이스
- ✅ 근무 일정 관리
- ✅ 효율성과 공정성을 중심으로 한 스케줄링

**타겟 사용자:**
- 간호사 (Nurses)
- 병원 관리자

---

### 3.2 Dutying
**웹 기능:**
- ✅ 근무표 작성 (Duty schedule creation)
- ✅ 관리자용 인터페이스 (수간호사용)
- ✅ 근무표 저장 및 관리

**앱 기능:**
- ✅ 개인 일정 관리 (Personal schedule management)
- ✅ 근무 일정 조회
- ✅ 월별 근무 등록 현황 조회
- ✅ 동료의 근무 일정 조회
- ✅ 친구/동료와 일정 공유
- ✅ 개인 일정(외출, 외근 등) 병행 관리
- ✅ 그룹 초대 및 그룹별 일정 관리 (공세마리 병동, 경동 등)
- ✅ 일정별 필터링 (요약보기, 모아보기)

**타겟 사용자:**
- 일반 간호사 (App)
- 수간호사/관리자 (Web)

---

### 3.3 DutyPlanners
**핵심 기능:**
- ✅ 자동 스케줄 생성 (3시간 → 1분)
- ✅ 20개 이상의 제약 조건 자동 처리
- ✅ 공정한 근무 배정
- ✅ 모바일 앱 (어디서나 관리 가능)
- ✅ 간호사 원팀드 듀티 신청 반영
- ✅ 제약 조건 실시간 검증

**주요 가치 제안:**
- 시간 절감: 3시간 → 1분
- 공정성 보장: 20개 이상의 제약 조건 자동 고려
- 만족도 향상: 원팀드 듀티 신청 반영으로 이직률 감소
- 편리한 관리: 모바일에서 언제 어디서나 관리

**통계:**
- 20+: 처리 가능한 제약 조건 개수
- 1분: 스케줄 생성 소요 시간
- 85%+: 원팀드 듀티 방영율

## 4. 근무일정표 작성을 위한 세부 기능

### 4.1 기본 입력 설정

#### 4.1.1 인력 구성
- **필수 입력항목:**
  - 총 간호사 수
  - 간호사 이름/ID
  - 부서/병동 정보 (optional)

#### 4.1.2 기간 설정
- **필수 입력항목:**
  - 월 선택 (예: 2026년 02월)
  - 스케줄 작성 기간

#### 4.1.3 근무 유형 정의
- **표준 근무 유형:**
  - D (Day/낮번): 주간 근무
  - E (Evening/저녁번): 오후~저녁 근무
  - N (Night/밤번): 야간 근무
  - O (Off/휴무): 당일 비근무
  - X (기타): 연차, 교육 등

---

### 4.2 근무 순서 설정 (근무 순환 규칙)

#### 4.2.1 근무 전환 규칙 (Shift Transition Rules)
MakeDuty 예시:
- ☑️ 야간 근무 후 주간 근무 허용 (ND 허용)
- ☑️ 야간 근무 후 저녁 근무 허용 (NE 허용)
- ☑️ 저녁 근무 후 주간 근무 허용 (ED 허용) - **활성화**
- ☑️ 야간 휴무-주간 연속 근무 허용 (NOD 허용)

#### 4.2.2 연속 근무 제한
- 연속 근무일 수 제한 해제 옵션
- 연속 야간 근무일 수 제한 해제 옵션

---

### 4.3 근무표 작성 규칙 설정

#### 4.3.1 연속 근무 제약
MakeDuty 예시:
- 최대 연속 근무 수: 5일 (기본값)
- 최대 연속 야간근무(N) 수: 3일 (기본값)
- 일일 휴무일 수: 8일 (기본값)

#### 4.3.2 일자별 필요 인원 설정

**평일 (Weekday):**
- 평일 주간 근무 인원(D): 4명
- 평일 저녁 근무 인원(E): 4명
- 평일 야간 근무 인원(N): 4명

**주말 (Weekend):**
- 주말 주간 근무 인원(D): 3명
- 주말 저녁 근무 인원(E): 3명
- 주말 야간 근무 인원(N): 3명

#### 4.3.3 특수일 설정
- 공휴일 설정
- 휴무일 설정
- 교육일/연차 설정

---

### 4.4 제약 조건 (Constraints)

#### 4.4.1 하드 제약 (Hard Constraints)
DutyPlanners에서 언급하는 20개 이상의 제약 조건 중 주요 항목:

1. **최대 연속 근무일 제한**
   - 예: 최대 5일 연속 근무

2. **밤번 후 최소 휴식**
   - 예: Night shift 후 최소 휴식 시간 보장

3. **병동별 근무별 최소 인원 보장**
   - 예: 매일 D조 4명, E조 4명, N조 4명

4. **야간 근무 제약**
   - 최대 연속 야간 근무: 3일
   - Night shift 후 다음 날 가능한 shift 제한

5. **휴무일 보장**
   - 월간 최소 휴무일 수

---

#### 4.4.2 소프트 제약 (Soft Constraints)

1. **개인 선호도**
   - 특정 날짜 휴무 요청
   - 특정 shift 선호도

2. **월간 근무 유형별 균등 배분**
   - D, E, N 근무 횟수 균등화
   - 개별 간호사별 공정한 분배

3. **동일 간호사의 연속 야간 근무 제한**
   - 연속 Night shift 횟수 제한
   - 야간 근무 집중 방지

4. **개인 일정 반영**
   - 교육, 연차 등 사전 등록된 일정

---

### 4.5 스케줄 생성 및 최적화

#### 4.5.1 자동 생성 프로세스
- 입력된 모든 조건을 기반으로 자동 배정
- 제약 조건 위반 여부 실시간 검증

#### 4.5.2 결과 표시
- **스케줄 테이블:**

  - 셀 값: D, E, N, O 등의 근무 유형

- **통계 정보:**
  - 각 간호사의 근무 유형별 횟수
  - 월간 휴무일 수
  - 연속 근무 현황

## 5. 분석 결과 및 개발 시 고려사항

### 5.1 공통적으로 중요한 기능
1. **자동 스케줄 생성**: 모든 서비스의 핵심 가치
2. **제약 조건 관리**: 20개 이상의 복잡한 제약 처리
3. **모바일 지원**: 관리자와 간호사 모두 필요
4. **사용자 친화적 UI**: 복잡한 설정을 단순하게 표현
5. **실시간 검증**: 규칙 위반 여부 즉시 피드백

### 5.2 사용자별 요구사항 차이
- **관리자**: 효율성(시간 단축), 공정성, 일괄 관리
- **간호사**: 개인 일정 관리, 선호도 반영, 투명성

### 5.3 기술적 고려사항
- **빠른 알고리즘**: 1분 내 대규모 스케줄 생성
- **제약 충족 검증**: 하드 제약은 100% 충족, 소프트 제약은 페널티 시스템
- **확장성**: 병원 규모에 따른 간호사 수 변화 대응
- **다중 플랫폼**: Web과 Mobile 동시 지원

### 5.4 UI/UX 설계 원칙
- 복잡한 설정을 단계별로 분리
- 시각적 피드백 (색상 코딩: D-파란색, E-초록색, N-보라색, O-회색)
- 결과물의 즉시 확인 가능
- 직관적인 수정 인터페이스 (드래그, 클릭 등)
