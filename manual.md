## 1. 시작하기

### 1.1 설치 및 실행

NurseSch는 **Python FastAPI 백엔드**와 **React 프론트엔드**로 구성되어 있습니다.

```bash
# 1. 프론트엔드 의존성 설치
npm install

# 2. Python 백엔드 의존성 설치
C:/ProgramData/Anaconda3/python.exe -m pip install fastapi uvicorn sqlalchemy python-jose bcrypt python-multipart openpyxl

# 3. 시드 데이터 생성 (초기 1회)
cd backend_py && C:/ProgramData/Anaconda3/python.exe seed.py && cd ..

# 4. 개발 서버 실행 (백엔드 + 프론트엔드 동시 실행)
npm run dev
```

실행 후 브라우저에서 **http://localhost:5173** 으로 접속합니다.

> **주의:** `npm run dev` 명령은 반드시 **프로젝트 루트 폴더** (`nursesch/`)에서 실행해야 합니다.
> 이 명령은 백엔드 서버(port 3000)와 프론트엔드 서버(port 5173)를 **동시에** 실행합니다.
> 하나만 실행하면 서비스가 정상 동작하지 않습니다.

### 1.2 서버 구성

NurseSch는 **2개의 서버**로 구성됩니다. 두 서버가 모두 실행 중이어야 정상 동작합니다.

| 서버 | 포트 | 역할 | 접속 URL |
|------|------|------|---------|
| **프론트엔드** (Vite) | 5173 | 사용자 화면 (React) | http://localhost:5173 (브라우저 접속) |
| **백엔드** (FastAPI + uvicorn) | 3000 | REST API 서버 | http://localhost:3000/api/* (자동 연결) |

> 사용자는 항상 **http://localhost:5173** 으로 접속합니다.
> 프론트엔드가 백엔드 API를 자동으로 프록시(중계)하므로 백엔드 URL에 직접 접속할 필요가 없습니다.

#### 개별 실행 (선택)

터미널 2개를 열어 각각 실행하는 것도 가능합니다:

```bash
# 터미널 1 — 백엔드
npm run dev:backend

# 터미널 2 — 프론트엔드
npm run dev:frontend
```

#### 정상 실행 확인

서버가 올바르게 실행되면 콘솔에 다음과 유사한 메시지가 표시됩니다:

```
VITE v6.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
INFO:     Uvicorn running on http://0.0.0.0:3000
```

두 메시지가 모두 표시되어야 합니다.

#### 서버 종료

**`npm run dev`로 동시 실행한 경우:**
- 서버가 실행 중인 터미널에서 **`Ctrl + C`** 를 누르면 백엔드(uvicorn)와 프론트엔드(Vite)가 **모두 동시에 종료**됩니다.

**개별 실행(`dev:backend`, `dev:frontend`)한 경우:**
- 각 터미널에서 **`Ctrl + C`** 를 눌러 개별 종료합니다.

**비정상 종료 후 포트가 사용 중일 때:**

서버를 정상 종료하지 못한 경우 "포트가 이미 사용 중입니다" 오류가 발생할 수 있습니다. 이때는 해당 포트를 사용하는 프로세스를 찾아 강제 종료합니다.

```bash
# Windows (CMD 또는 PowerShell)
# 1. 포트를 사용하는 프로세스 PID 확인
netstat -ano | findstr :3000    # 백엔드
netstat -ano | findstr :5173    # 프론트엔드

# 2. 해당 PID 강제 종료
taskkill /PID <PID번호> /F

# Git Bash / MSYS2
# 1. 포트를 사용하는 프로세스 확인
netstat -ano | grep :3000
netstat -ano | grep :5173

# 2. 해당 PID 강제 종료
taskkill //PID <PID번호> //F
```

### 1.3 초기 계정 정보

시드 데이터로 생성되는 기본 계정입니다.

| 구분 | 이메일 | 비밀번호 | 직급 | 역할 |
|------|--------|---------|------|------|
| 관리자 (수간호사) | `admin@hospital.com` | `password123` | HN (수간호사) | ADMIN |
| 책임간호사 1~2 | `nurse1~2@hospital.com` | `password123` | CN (책임간호사) | — |
| 평간호사 3~8 | `nurse3~8@hospital.com` | `password123` | RN (평간호사) | — |
| 보조간호사 9 | `nurse9@hospital.com` | `password123` | AN (보조간호사) | — |

- 관리자 계정은 **김수간** (내과 5병동 수간호사)으로 등록되어 있습니다.
- 간호사는 총 **9명**이 사전 등록되어 있습니다 (관리자 포함 총 10명).

### 1.4 데이터베이스

- SQLite 파일 기반 데이터베이스를 사용합니다.
- DB 파일 위치: `backend_py/nursesch.db`
- 별도의 DB 서버 설치가 필요 없습니다.
- 데이터를 초기화하려면 `nursesch.db` 파일을 삭제한 뒤 `seed.py`를 다시 실행하면 됩니다.

## 2. 로그인

1. 브라우저에서 **http://localhost:5173** 에 접속하면 **로그인 화면**이 표시됩니다.
   - 보라색 그라데이션 배경에 흰색 로그인 카드가 표시됩니다.
   - "NurseSch" 로고와 "간호사 근무표 자동 생성 서비스" 부제목이 보입니다.
2. 이메일과 비밀번호를 입력한 뒤 **로그인** 버튼을 클릭합니다.
3. 로그인에 성공하면 **대시보드**로 이동하며, 직급에 따라 다른 메뉴가 표시됩니다.
   - **관리자 (HN)**: 대시보드, 간호사 관리, 근무 규칙, 근무표, 근무 요청, 통계
   - **간호사 (CN/RN/AN)**: 대시보드, 근무표, 근무 요청, 통계

> 로그아웃하려면 좌측 사이드바 하단의 사용자 이름 아래 **로그아웃** 버튼을 클릭합니다.

## 3. 화면 구성

로그인 후 화면은 크게 두 영역으로 나뉩니다.

```
┌──────────┬────────────────────────────────┐
│          │                                │
│ 사이드바  │         메인 콘텐츠 영역         │
│          │                                │
│ - 로고    │   (선택한 메뉴의 내용 표시)      │
│ - 메뉴    │                                │
│ - 사용자  │                                │
│ - 로그아웃│                                │
│          │                                │
└──────────┴────────────────────────────────┘
```

**사이드바 메뉴:**

| 메뉴 | 관리자(HN) | 간호사(CN/RN/AN) | 설명 |
|------|:----------:|:----------------:|------|
| 대시보드 | O | O | 병동 현황 요약, 최근 근무표 목록 |
| 간호사 관리 | O | - | 간호사 등록/수정/삭제, 직급 관리 |
| 근무 규칙 | O | - | 제약 조건, Charge/Action 인원, 전환 규칙 설정 |
| 근무표 | O | O | 근무표 자동 생성/조회/수정 |
| 근무 요청 | O | O | 근무 선호도 요청 및 연차 관리 |
| 통계 | O | O | 간호사별 근무 통계 대시보드 |

## 4. 관리자 기능

### 4.1 대시보드

로그인 직후 표시되는 홈 화면입니다.

**표시 정보 (상단 카드 4개):**
- 소속 병동명
- 등록된 간호사 수
- 전체 근무표 수
- 작성중(DRAFT) 근무표 수 (1개 이상이면 주황색 강조)

**빠른 액션 버튼:**
- `간호사 관리` — 간호사 관리 화면으로 이동
- `근무 규칙 설정` — 근무 규칙 설정 화면으로 이동
- `근무표 생성` — 근무표 화면으로 이동
- `근무 요청 관리` — 근무 요청 화면으로 이동

**최근 근무표 목록:**
- 최근 5개 근무표의 연월, 상태, 생성일이 표시됩니다.
- **보기** 버튼을 클릭하면 해당 근무표 상세 화면으로 이동합니다.
- 상태 뱃지: `작성중` (파란 배경) / `확정` (초록 배경)

### 4.2 간호사 관리

사이드바에서 **간호사 관리**를 클릭합니다.

#### 간호사 목록 조회

소속 병동의 전체 간호사 목록이 테이블로 표시됩니다.

| 열 | 설명 |
|----|------|
| 순서 | 정렬 순서 번호 (드래그로 변경 가능) |
| 이름 | 간호사 이름 (클릭하여 정렬) |
| 이메일 | 로그인 이메일 (클릭하여 정렬) |
| 직급 | HN/CN/RN/AN 뱃지 (클릭하여 정렬) |
| 나이트 전담 | 전담 지정 여부 (보라색 ● 전담 표시) |
| 월 연차 | 월간 기본 연차 일수 |
| 관리 | 수정/삭제 버튼 |

**정렬 기능:**
- 이름, 이메일, 직급 열 헤더를 클릭하면 해당 기준으로 정렬됩니다.
- 관리자가 지정한 정렬 순서(sort_order)가 모든 정렬보다 우선합니다.

**드래그 앤 드롭 순서 변경:**
- 테이블 행을 드래그하여 원하는 위치에 놓으면 순서가 변경됩니다.
- 변경된 순서는 즉시 서버에 저장됩니다.
- 지정 순서는 근무표에서 간호사 표시 순서에 반영됩니다.

#### 간호사 추가

1. 우측 상단의 **+ 간호사 추가** 버튼을 클릭합니다.
2. 팝업 창에서 다음 정보를 입력합니다.

| 항목 | 설명 | 필수 |
|------|------|:----:|
| 이름 | 간호사 이름 | O |
| 이메일 | 로그인용 이메일 (중복 불가) | O |
| 비밀번호 | 초기 비밀번호 | O |
| 직급 | HN (수간호사) / CN (책임간호사) / RN (평간호사) / AN (보조간호사) | O |
| 정렬 순서 | 0=자동, 1 이상 = 수동 지정 | - |
| 월 연차 수 | 월간 기본 연차 일수 (기본값: 1일) | - |
| 나이트 전담 | 체크 시 N/O만 배정 | - |

3. **등록** 버튼을 클릭하면 등록이 완료됩니다.

> 나이트 전담을 지정하면 해당 간호사는 자동 생성 시 나이트(N)와 오프(O)만 배정됩니다.

#### 간호사 수정

1. 목록에서 해당 간호사의 **수정** 버튼을 클릭합니다.
2. 이름, 이메일, 직급, 순서, 월 연차, 나이트 전담 여부를 수정할 수 있습니다.
3. 비밀번호는 변경 시만 입력합니다 (빈칸이면 기존 비밀번호 유지).
4. **수정 완료** 버튼을 클릭하면 변경이 저장됩니다.

#### 간호사 삭제

1. 목록에서 해당 간호사의 **삭제** 버튼을 클릭합니다.
2. 확인 대화상자에서 **확인**을 클릭하면 삭제됩니다.

### 4.3 근무 규칙 설정

사이드바에서 **근무 규칙**을 클릭합니다.

화면은 **좌측 패널**과 **우측 패널** 2개 영역으로 구성됩니다.

#### 좌측 패널 — 기본 설정

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 최대 연속 근무일 수 | 간호사가 연속으로 근무할 수 있는 최대 일수 | 5일 |
| 최대 연속 나이트 수 | 나이트를 연속으로 할 수 있는 최대 일수 | 3일 |
| 월간 최소 휴무일 수 | 한 달에 보장되어야 하는 최소 휴무일 | 8일 |

#### 좌측 패널 — 근무 전환 금지 패턴

토글 스위치로 특정 근무 전환 패턴의 허용 여부를 설정합니다.

| 토글 | 설명 | 기본값 |
|------|------|--------|
| N → 데이 허용 | 나이트 다음 날 데이 근무 허용 | OFF (금지) |
| N → 이브닝 허용 | 나이트 다음 날 이브닝 근무 허용 | OFF (금지) |
| N → 오프 → 데이 허용 | 나이트→오프→데이 패턴 허용 | OFF (금지) |
| AN 주말 자동 Off | 보조간호사(AN) 토/일 자동 휴무 | ON (활성) |

> 기본적으로 근무 전환 규칙이 **금지**로 설정되어 있습니다. 병동 사정에 따라 허용이 필요하면 토글을 ON으로 변경합니다.
> AN 주말 자동 Off를 끄면 보조간호사도 주말 근무에 배정될 수 있습니다.

#### 우측 패널 — Charge/Action 인원 설정

각 근무(D/E/N)에 필요한 **Charge** (책임) 인원과 **Action** (실무) 인원을 설정합니다.

- **Charge**: HN (수간호사), CN (책임간호사) 직급
- **Action**: RN (평간호사) 직급

**평일 인원 설정:**

| 근무 | Charge (HN/CN) | Action (RN) |
|------|:--------------:|:-----------:|
| 데이 (D) | 기본 1명 | 기본 1명 |
| 이브닝 (E) | 기본 1명 | 기본 1명 |
| 나이트 (N) | 기본 0명 | 기본 1명 |

**주말/공휴일 인원 설정:** (평일과 동일 구조)

> **중요:** Charge 인원 설정 시, 해당 직급(HN/CN)의 실제 간호사 수를 고려하세요.
> 예를 들어 HN 1명 + CN 2명 = 총 Charge 3명인데, 평일 D=1, E=1, N=1로 설정하면 매일 3명 전원이 근무해야 하므로 휴무를 줄 수 없습니다. 이 경우 나이트 Charge를 0으로 설정하는 것을 권장합니다.

#### 나이트 전담 간호사 표시

우측 패널 하단에 현재 나이트 전담으로 지정된 간호사 목록이 표시됩니다.

- 나이트 전담 간호사가 없으면 안내 메시지가 표시됩니다.
- 전담 지정은 **간호사 관리** 화면에서 설정합니다.

#### 저장

설정을 변경한 뒤 우측 상단의 **저장** 버튼을 클릭합니다. 저장이 완료되면 "규칙이 저장되었습니다." 메시지가 표시됩니다.

> 근무 규칙은 병동별로 독립 관리됩니다. 설정 변경은 다음 번 근무표 자동 생성부터 적용됩니다.

---

### 4.4 근무표 자동 생성

사이드바에서 **근무표**를 클릭합니다.

#### 생성 절차

1. **대상 연월 선택**: 근무표를 생성할 년/월을 선택합니다 (날짜 선택기).
2. **자동 생성** 버튼을 클릭합니다.
3. 시스템이 SA(Simulated Annealing) 알고리즘을 실행하여 최적의 근무표를 생성합니다.
4. 생성이 완료되면 결과가 화면에 표시됩니다.

**생성 결과 표시:**
- **초록색 배너**: 하드 위반 0건 — 정상 생성됨
- **빨간색 배너**: 하드 위반 N건 — 위반 사항과 점수 표시 (상위 5건까지)

**생성 시 자동으로 반영되는 사항:**
- 근무 규칙 설정의 모든 제약 조건 (인원, 전환 패턴, 연속 근무 등)
- 승인된 간호사 근무 요청
- 간호사별 연차/희망 휴일 설정
- 나이트 전담/AN 주말 자동 Off 설정

**생성 소요 시간:**
- 간호사 10명 기준: 약 10~15초
- 간호사 20명 이상: 약 15~30초
- 버튼이 "생성 중... (10~30초 소요)"로 변경됩니다.

> 같은 연월에 대해 여러 번 생성할 수 있습니다.

### 4.5 근무표 조회 및 수정

근무표 목록에서 **보기** 버튼을 클릭하면 상세 화면으로 이동합니다.

#### 근무표 테이블 뷰

**상단 영역:**
- 좌측: `← 목록` 버튼 + 연월 제목 (예: "2026-02 근무표") + 상태 뱃지 (`작성중` 또는 `확정`)
- 우측: `근무표 확정` 버튼 (작성중일 때만) + `Excel 내보내기` 버튼

**테이블 영역:**

```
간호사   1   2   3   4   5   6   7  ...  27  28   D   E   N   M   O   Y
         일  월  화  수  목  금  토       금  토
김수간   E   O   O   D   D   E   D  ...   O   O    9   5   4   0  10   0
강도현   N   N   O   E   E   D   N  ...   E   D    8   6   5   0   9   0
박지민   D   D   E   O   O   O   D  ...   E   D    7   7   5   0   9   0
...
```

- **행**: 간호사 이름
- **열**: 날짜 (1일~말일), 각 날짜 아래에 요일 표시
- **셀**: 근무 유형 코드가 색상 배경으로 표시
- **우측 통계열**: 간호사별 D / E / N / M / O / Y 횟수 합산
- **주말**: 날짜 헤더 배경색이 다르게 표시
- 테이블이 화면보다 넓으면 **가로 스크롤**이 가능합니다

**색상 코딩:**

| 코드 | 명칭 | 배경색 | 글자색 |
|------|------|--------|--------|
| `D` | 데이 | 연한 파랑 | 진한 파랑 |
| `E` | 이브닝 | 연한 초록 | 진한 초록 |
| `N` | 나이트 | 연한 보라 | 진한 보라 |
| `M` | 미드 | 연한 청색 | 진한 청색 |
| `O` | 오프 | 연한 회색 | 회색 |
| `Y` | 연차 | 연한 노랑 | 진한 노랑 |
| `X` | 기타 | 흰색 | 회색 |

**수동 수정 표시:**
- 관리자가 수동으로 수정한 셀은 **굵은 글씨**로 표시됩니다.

#### 개별 셀 수정

> 근무표가 **작성중(DRAFT)** 상태이고, **관리자**로 로그인했을 때만 수정 가능합니다.

1. 변경하고 싶은 셀을 **클릭**합니다.
2. 셀이 드롭다운으로 변경되며 근무 유형 목록이 표시됩니다.
   - D (데이), E (이브닝), N (나이트), M (미드), O (오프), Y (연차), X (기타)
3. 원하는 근무 유형을 선택하면 즉시 변경됩니다.
4. 변경 후 **제약 조건이 자동으로 재검증**됩니다.

**제약 위반 표시:**
- 하드 제약을 위반하는 셀에는 **빨간색 테두리 (2px 빨간 실선)**가 표시됩니다.
- 화면 상단에 "하드 제약 위반 N건" 경고 배너가 표시됩니다 (상위 3건의 메시지와 추가 건수).
- 화면 하단의 **제약 위반 상세** 패널에서 모든 위반 사항을 확인할 수 있습니다 (최대 20건 표시).
  - 하드 제약 위반: 빨간색 텍스트
  - 소프트 제약 위반: 주황색 텍스트

### 4.6 근무표 확정 및 내보내기

#### 근무표 확정

1. 근무표 상세 화면에서 우측 상단의 **근무표 확정** 버튼을 클릭합니다.
2. "근무표를 확정하시겠습니까? 확정 후에는 수정할 수 없습니다." 확인 대화상자가 표시됩니다.

4. 파일에는 간호사별 월간 근무 일정과 근무 유형별 횟수 통계가 포함됩니다.

### 4.7 근무 요청 및 연차 관리

사이드바에서 **근무 요청**을 클릭합니다.

화면 상단에서 **대상 연월**을 선택할 수 있으며, **2개의 탭**으로 구성됩니다.

#### 탭 1: 근무 요청

**관리자 직접 근무/휴일 지정 (상단 폼):**

관리자는 특정 간호사에게 직접 근무나 휴일을 지정할 수 있습니다.

1. 간호사 선택 (드롭다운)
2. 날짜 선택
3. 근무 유형 선택 (D/E/N/M/O/Y/X)
4. **지정** 버튼 클릭

> 관리자 지정 요청은 자동으로 승인 처리됩니다.

**요청 목록 (테이블):**

| 열 | 설명 |
|----|------|
| 간호사 | 요청한 간호사 이름 |
| 날짜 | 요청 대상 날짜 |
| 근무 | 요청한 근무 유형 (색상 뱃지) |
| 구분 | "관리자 지정" 또는 "간호사 요청" |
| 상태 | 대기 (주황) / 승인 (초록) / 반려 (빨강) |
| 사유 | 간호사가 입력한 사유 |
| 관리 | 승인/반려/삭제 버튼 |

**승인 / 반려:**
- **대기(PENDING)** 상태의 요청에 대해 **승인** 또는 **반려** 버튼이 표시됩니다.
- **승인**: 자동 생성 시 해당 요청이 고정 배정으로 반영됩니다.
- **반려**: 해당 요청이 무시됩니다.

#### 탭 2: 연차 / 희망 휴일

관리자는 간호사별 월간 연차 수와 희망 휴일을 관리할 수 있습니다.

| 열 | 설명 |
|----|------|
| 간호사 | 간호사 이름 |
| 직급 | HN/CN/RN/AN |
| 월 연차수 | 해당 월 연차 일수 (수정 가능) |
| 희망 휴일 | YYYY-MM-DD 형식, 콤마로 구분 (수정 가능) |
| 총 비근무일수 | 주말 + 공휴일 + 연차 합산 (자동 계산) |
| 관리 | 수정/저장/취소 버튼 |

> **총 비근무일수** = 해당 월 주말일수 + 공휴일수(주말 제외) + 개인 연차수
> 이 일수가 근무표 자동 생성 시 해당 간호사의 목표 비근무일수로 반영됩니다.

## 7. 간호사 직급 안내

| 직급 코드 | 명칭 | 스케줄링 역할 | 특이사항 |
|-----------|------|-------------|----------|
| `HN` | 수간호사 | Charge (책임) | 관리자 권한 (ADMIN) |
| `CN` | 책임간호사 | Charge (책임) | — |
| `RN` | 평간호사 | Action (실무) | — |
| `AN` | 보조간호사 | — | 미드(M) 전담, 주말 자동 Off (설정 가능) |

**Charge vs Action:**
- **Charge** (HN, CN): 각 근무(D/E/N)에 최소 Charge 인원이 배정되어야 합니다. 근무조의 책임자 역할입니다.
- **Action** (RN): 각 근무에 최소 Action 인원이 배정되어야 합니다. 실무 담당입니다.
- **AN**: 평일 미드(M) 근무가 기본 배정되며, 주말은 AN 주말 자동 Off 설정에 따라 자동 휴무 처리됩니다.

**나이트 전담:**
- 간호사를 "나이트 전담"으로 지정하면 나이트(N)와 오프(O)만 배정됩니다.
- 나이트 전담 간호사가 있으면 일반 간호사의 나이트 부담이 줄어듭니다.

## 8. 제약 조건 안내

자동 생성 알고리즘(SA)이 적용하는 제약 조건입니다.

### 하드 제약 (Hard Constraints) — 위반 시 빨간색 경고

위반 시 근무표가 유효하지 않으며, 건당 1000점 페널티가 부과됩니다.

| 코드 | 제약 | 설명 |
|------|------|------|
| HC-1 | 근무조별 인원 충족 | 매일 각 근무(D/E/N)에 설정된 최소 Charge/Action 인원이 배정되어야 합니다 |
| HC-2 | 연속 근무 제한 | 설정된 최대 연속 근무일을 초과할 수 없습니다 (기본 5일) |
| HC-3 | 금지 전환 패턴 | N→D, N→E, N→O→D 등 설정된 금지 패턴이 존재하면 안 됩니다 |
| HC-4 | 나이트 연속 제한 | 나이트 단독 1일 금지, 연속 4일 이상 금지 (2~3일만 허용) |
| HC-5 | 나이트 전담 준수 | 나이트 전담 간호사는 N/O/Y/X만 배정 가능 |
| HC-6 | AN 주말 자동 Off | AN 간호사 주말 근무 금지 (규칙 설정에서 비활성화 가능) |

### 소프트 제약 (Soft Constraints) — 위반 시 가중 페널티

위반 시 페널티 점수가 부과되며, 알고리즘이 최소화하려고 합니다.

| 코드 | 제약 | 설명 |
|------|------|------|
| SC-1 | 요청 반영 | 승인된 간호사 근무 요청을 가능한 한 반영합니다 |
| SC-2 | 균등 배분 | 간호사 간 총 근무일수를 균등하게 배분합니다 (표준편차 × 50 페널티) |

## 9. 자주 묻는 질문 (FAQ)

### Q. 근무표 생성 시 "병동 또는 규칙이 존재하지 않습니다" 오류가 뜹니다.
**A.** 병동에 근무 규칙이 설정되어 있지 않을 수 있습니다. **근무 규칙** 메뉴에서 규칙을 확인하고 **저장**을 클릭해 주세요.

### Q. 자동 생성했는데 빨간색 위반 표시가 있습니다.
**A.** 주로 HC-1 (인원 부족)이 원인입니다. 간호사 수가 최소 인원 요구사항에 비해 부족할 수 있습니다.

**해결 방법:**
1. **간호사 추가**: 충분한 수의 간호사를 등록합니다.
2. **인원 요구 수 조정**: 근무 규칙에서 Charge/Action 인원을 줄입니다.
3. **나이트 Charge 0으로 설정**: HN/CN이 3명 이하면 나이트 Charge를 0으로 설정하는 것을 권장합니다.

> 예: 평일 D(Charge=1, Action=1) + E(Charge=1, Action=1) + N(Charge=0, Action=1) = 최소 5명 근무 필요. 10명의 간호사가 있으면 충분합니다.

### Q. AN(보조간호사)에게 주말 근무를 배정하고 싶습니다.
**A.** **근무 규칙** 화면에서 "AN 주말 자동 Off" 토글을 OFF로 변경한 뒤 **저장**을 클릭하세요. 다음 자동 생성부터 적용됩니다.

### Q. 특정 간호사의 요청이 반영되지 않았습니다.
**A.** 근무 요청은 고정 배정으로 처리되지만, 다른 하드 제약과 충돌하는 경우 SA 알고리즘이 최적화 과정에서 변경할 수 있습니다. 반드시 반영해야 한다면 생성 후 수동으로 셀을 수정하세요.

### Q. N→D 전환을 허용하고 싶습니다.
**A.** **근무 규칙** 화면에서 "N → 데이 허용" 토글을 ON으로 변경한 뒤 **저장**을 클릭하세요. 다음 자동 생성부터 적용됩니다.

### Q. 확정한 근무표를 다시 수정하고 싶습니다.
**A.** 확정(CONFIRMED) 상태의 근무표는 직접 수정할 수 없습니다. 같은 연월로 새로 자동 생성하여 별도의 근무표를 만들 수 있습니다.

### Q. 나이트 전담 간호사를 지정하고 싶습니다.
**A.** **간호사 관리** 화면에서 해당 간호사의 **수정** 버튼을 클릭한 뒤, "나이트 전담 지정" 체크박스를 체크하고 저장합니다. 지정된 간호사는 자동 생성 시 나이트(N)와 오프(O)만 배정됩니다.

### Q. 간호사별 연차 일수를 다르게 설정하고 싶습니다.
**A.** **근무 요청** 화면에서 **연차 / 희망 휴일** 탭으로 이동합니다. 각 간호사의 "월 연차수"를 개별적으로 수정할 수 있습니다. 기본값은 간호사 등록 시 설정한 "월 연차 수"입니다.

### Q. Excel로 내보낸 파일에 어떤 정보가 포함되나요?
**A.** 간호사 이름, 1일~말일 근무 유형, 근무 유형별 횟수 통계가 포함됩니다. 파일명은 `schedule_YYYY-MM.xlsx` 형식입니다.

### Q. 간호사 계정의 비밀번호를 변경하고 싶습니다.
**A.** **간호사 관리** 화면에서 해당 간호사의 **수정** 버튼을 클릭하고 새 비밀번호를 입력한 뒤 저장합니다. 빈칸으로 두면 기존 비밀번호가 유지됩니다.

### Q. 데이터를 초기화하고 싶습니다.
**A.** `backend_py/nursesch.db` 파일을 삭제한 뒤 `cd backend_py && C:/ProgramData/Anaconda3/python.exe seed.py`를 실행하면 초기 데이터로 복원됩니다.

### Q. http://localhost:5173 접속 시 "연결을 거부했습니다" 오류가 뜹니다.
**A.** 프론트엔드 서버(Vite)가 실행되지 않은 상태입니다. **프로젝트 루트에서 `npm run dev`를 실행**하면 백엔드와 프론트엔드가 동시에 시작됩니다. 프론트엔드만 별도로 실행하려면 `npm run dev:frontend`를 사용하세요.

### Q. http://localhost:3000 접속 시 아무 페이지도 표시되지 않습니다.
**A.** 정상 동작입니다. 백엔드 서버는 API 전용 서버로 `/api/*` 경로만 제공합니다. **브라우저에서는 반드시 http://localhost:5173 으로 접속**하세요.
