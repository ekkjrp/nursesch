# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 프로젝트 개요

`nursesch/` 폴더는 **병원 간호사를 위한 근무 일정표(근무표)를 자동으로 생성해 주는 웹서비스**를 개발하기 위한 프로젝트 폴더이다. 알고리즘 설계는 `간호사_근무표_생성을_위한_알고리즘.pdf`에 정의되어 있으며, 스케줄링 로직 구현 전 반드시 참고해야 한다.

## 요구사항 문서

- 기능 요구사항은 `./requirements.md` 파일에 별도로 정리한다.
- **코드를 생성할 경우 반드시 `requirements.md`의 요구사항에 맞게 작성**해야 한다.
- 새로운 기능 추가 전, `requirements.md`를 먼저 확인하고 해당 요구사항을 충족하는지 검토한다.

## 개발 명령어

```bash
# 의존성 설치
npm install

# DB 생성 및 마이그레이션
cd backend && npx prisma generate && npx prisma db push

# 시드 데이터 생성
cd backend && npx tsx src/seed.ts

# 개발 서버 실행 (백엔드 + 프론트엔드 동시)
npm run dev

# 백엔드만 실행 (port 3000)
npm run dev:backend

# 프론트엔드만 실행 (port 5173)
npm run dev:frontend

# 전체 테스트 실행
npm test

# 단일 테스트 파일 실행
npm test -- backend/src/engine/__tests__/validator.test.ts
```

## 아키텍처 개요

> 프로젝트 구조 확정 후 업데이트한다.

핵심 레이어:
- **UI Layer** — 근무표 입력(간호사 목록, 선호도, 기간), 결과 표시
- **Scheduling Engine** — 제약 조건 기반 자동 배정 알고리즘 (`간호사_근무표_생성을_위한_알고리즘.pdf` 참고)
- **Constraint Validator** — 하드/소프트 제약 검증 모듈 (알고리즘 구현과 분리)
- **Persistence Layer** — 근무표, 간호사 프로필, 설정 저장

## 도메인 컨텍스트

### 근무 유형 (Shift Types)

| 코드 | 명칭 | 비고 |
|------|------|------|
| `D`  | 낮번 (Day) | 주간 근무 |
| `E`  | 저녁번 (Evening) | 오후~저녁 근무 |
| `N`  | 밤번 (Night) | 야간 근무 |
| `O`  | 휴무 (Off) | 당일 비근무 |
| `X`  | 기타 (연차, 교육 등) | 필요 시 확장 |

### 제약 조건 분류

**하드 제약 (Hard Constraints)** — 반드시 충족, 위반 시 스케줄 무효:
- 최대 연속 근무일 제한
- 밤번 후 최소 휴식 시간 (법정 요건)
- 병동별 근무별 최소 인원 보장

**소프트 제약 (Soft Constraints)** — 가능한 충족, 위반 시 페널티 부과:
- 간호사 개인 선호도 (특정 날짜 휴무 요청 등)
- 월간 근무 유형별 균등 배분
- 동일 간호사의 연속 야간 근무 횟수 제한

### 알고리즘 참고 자료

`간호사_근무표_생성을_위한_알고리즘.pdf` — 스케줄링 엔진 구현 전 반드시 숙지. 주요 내용:
- 초기 배정 전략 및 탐색 알고리즘
- 하드/소프트 제약 정의 및 가중치
- 최적화 목표 함수

## 코드 작성 규칙

### 언어 및 주석
- 코드 식별자(변수명, 함수명 등)는 영문을 사용한다.
- 주석은 **영문과 한글을 적절히 혼용**하여 코드의 의도와 비즈니스 로직을 명확히 설명한다.
- 복잡한 스케줄링 규칙, 제약 조건, 도메인 지식은 한글 주석으로 설명한다.

### 코드 스타일
- 변수명, 함수명은 그 역할이 명확히 드러나도록 서술적으로 작성한다.
- 한 함수는 하나의 명확한 역할만 수행하도록 작성한다.

### 테스트
- 단계별 테스트: 단위 테스트(unit) → 통합 테스트(integration) → 시나리오 테스트(e2e) 순으로 검증한다.
- 스케줄링 알고리즘은 제약 조건 충족 여부를 검증하는 테스트 케이스를 반드시 포함한다.
